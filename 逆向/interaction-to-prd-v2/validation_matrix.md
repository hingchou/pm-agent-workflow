# 验证矩阵: `interaction-to-prd` (中文版)

使用这些测试用例来验证 Claude 是否在不中断的情况下正确执行技能。

| 测试用例 ID | 场景 | 输入代码片段 (心理检查) | 预期行为 (通过标准) |
| :--- | :--- | :--- | :--- |
| **TC-01** | **正常流程** | 代码包含清晰的 `onClick`, `zod` 校验, 和 `fetch('/api/user')`。 | 完美生成第 4、5、6 章。不提问。 |
| **TC-02** | **缺失后端** | 代码包含 `fetch('/api/complex-calc')` 但没有后端逻辑。 | **通过**: 为 `ComplexCalc` 生成 Tool 定义，并从请求参数推断输入。 <br>**失败**: 询问“这个 API 是做什么的？” |
| **TC-03** | **硬编码限制** | 代码包含 `const MAX_FILES = 3;` | **通过**: 输出包含“约束条件：最多 3 个文件 (来源：代码硬限制)”。 <br>**失败**: 询问“这是临时值吗？” |
| **TC-04** | **模糊 UI** | 代码包含 `<div>{status}</div>` 但没有标签。 | **通过**: 记录为“动态显示：显示 status 变量”。 <br>**失败**: 询问“这个标签应该叫什么？” |
| **TC-05** | **混合上下文** | 用户同时粘贴 React 组件和 Zod 上下文文件。 | **通过**: 将 Zod 中的约束合并到该组件的“数据输入”表中。 |

## 🧪 如何测试
1.  复制 `SKILL.md` 的内容。
2.  将其粘贴到新的 Claude 对话中以“安装”技能（或使用 Project 功能）。
3.  粘贴一段你的前端代码。
4.  输入指令："生成 PRD"。
5.  检查它是停下来问问题（失败）还是立即输出表格（通过）。
